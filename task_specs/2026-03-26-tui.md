## Description:
- create a simple tui that can be used for testing
- the tui should take a host, user_id, and topic_id as parameters 

## Technical Tasks and Spec:
- the tui should be tokio event loop based:
    - listen for input events from the tui framework or incoming websocket messages
    - on tui framework event
        - make the edit to the local crdt state 
        - emit an update operation
        - asynchronously write the update operation to the websocket
        - redraw the text area using the new crdt state
    - on websocket message
        - merge the update operation into the local crdt state
        - redraw the text area using the new crdt state
- [x] create a simple text editor tui app with no crdt or websocket capabilities
- [ ] link the document to textarea
    - [x] add a second pane of the text editor where the resolved crdt doc state is displayed
    - [x] each input event modifies the crdt, then the textarea gets redrawn with the new crdt state representation
        - the text area represents the document state as an array of strings. When a modification is made, we need to get the location of that modification so that we can intercept that modification and instead apply that modification to the document state
            - the textarea allows us to get the cursor as a tuple of integers (row, col)
            - convert from the cursor location to the global offset into the document
        - get the full document representation
        - update the contents of the textarea in place to have the document representation from the CRDT instead of applying the operation directly to the textarea
    - [x] after replacing the text area contents with the updated crdt representation, use the sticky cursor feature to:
        - [x] keep track of the users cursor in the YText
            - https://docs.rs/yrs/latest/yrs/#cursor-positioning
        - [x] move the cursor inside of the text area
            - https://docs.rs/tui-textarea/latest/tui_textarea/struct.TextArea.html#method.move_cursor
- [x] add the websocket connection
    - [x] use to tokio event loop to simultaneously listen for keyboard events and websocket messages
    - EventStream feature is not available by default, have to use this feature "event-stream"
    - for asynchronously selecting from the terminal event stream
        - https://github.com/crossterm-rs/crossterm/blob/master/examples/event-stream-tokio.rs
- [x] add stickiness to the cursor so that edits do not reset the cursor position
    - improve ux
    - this will give us a sticky index into the document
        - https://docs.yjs.dev/api/relative-positions
    - this allows us to set the cursor location in the textarea
        - textarea.move_cursor(CursorMove::Jump(row, col)

## Testing:
- run the backend in one terminal window
```bash
cd message_proxy
docker compose -fd docker-compose-sqlx.yml up
cargo run --bin message_proxy
```
- run the first client in a second terminal window
```bash
cd message_proxy
cargo run --bin tui -- ws://localhost:3000/ws/00000000-0000-0000-0000-000000000000/00000000-0000-0000-0000-000000000001
```
