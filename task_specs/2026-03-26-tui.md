## Description:
- create a simple tui that can be used for testing
- the tui should take a host, user_id, and topic_id as parameters 

## Technical Tasks and Spec:
- the tui should be tokio event loop based:
    - listen for input events from the tui framework or incoming websocket messages
    - on tui framework event
        - make the edit to the local crdt state 
        - emit an update operation
        - asynchronously write the update operation to the websocket
        - redraw the text area using the new crdt state
    - on websocket message
        - merge the update operation into the local crdt state
        - redraw the text area using the new crdt state
